库文件简介

库文件是许多含有实用功能的目标文件集合，每个目标文件存储的代码，并非完整的程序，而是一个个实用的功能模块。库文件的调用以 C 语言中的 printf() 输出函数为例，程序中只需引入 <stdio.h> 头文件，即可调用 printf() 函数。

调用库文件为什么还要牵扯到头文件呢？

因为头文件和库文件的区别在于：头文件只存储变量、函数或者类等这些功能模块的声明部分，库文件才负责存储各模块具体的实现部分。即所有的库文件都提供有相应的头文件作为调用它的接口。也就是说，库文件是无法直接使用的，只能通过头文件间接调用。

对于库文件，编译器提供有 2 种实现链接的方式，分别称为静态链接方式【静态链接库】和动态链接方式【动态链接库】。

静态链接库

在 Linux 发行版系统中，静态链接库文件的后缀名通常用 .a 表示；在 Windows 系统中，静态链接库文件的后缀名为 .lib。

静态链接库实现链接操作的方式很简单，即程序文件中哪里用到了库文件中的功能模块，GCC 编译器就会将该模板代码直接复制到程序文件的适当位置，最终生成可执行文件。

因为静态库将代码复制到指定位置，所以会带来如下的优缺点：

优点：生成的可执行文件不再需要任何静态库文件的支持就可以独立运行

缺点：多次调用库中的同一功能模块，则该模块代码势必就会被复制多次，造成代码冗余

动态链接库

在 Linux 发行版系统中，动态链接库的后缀名通常用 .so 表示；在 Windows 系统中，动态链接库的后缀名为 .dll。

动态链接库实现链接操作时，程序文件中哪里需要库文件的功能模块，GCC 编译器不会直接将该功能模块的代码拷贝到文件中，而是将功能模块的位置信息记录到文件中，直接生成可执行文件。采用动态链接库生成的可执行文件运行时，GCC 编译器会将对应的动态链接库一同加载在内存中，由于可执行文件中事先记录了所需功能模块的位置信息，所以在现有动态链接库的支持下，也可以成功运行。

由于动态链接库通过记录位置信息找到目标代码，所以带来如下的优缺点：

优点：真正的实现代码会在程序运行时被载入内存，这意味着，即便功能模块被调用多次，使用的是同一份代码，减少代码冗余

缺点：生成的可执行文件无法独立运行，必须借助相应的库文件

需要注意的是，并非任何一个源文件都可以被加工成静态链接库，其至少需要满足以下 2 个条件：

  • 源文件中只提供可以重复使用的代码，例如函数、设计好的类等，不能包含 main 主函数；
  • 源文件在实现具备模块功能的同时，还要提供访问它的接口，也就是包含各个功能模块声明部分的头文件。

注意，生成的 main.exe 通常无法直接执行，会报错显示执行过程中无法找到动态链接库【可以通过命令 ldd main.exe 验证】。运行由动态链接库生成的可执行文件时，必须确保程序在运行时可以找到这个动态链接库。常用的解决方案有如下几种：

  • 通过拷贝或者创建链接的方式，将链接库文件移动到标准库目录下（例如 /usr/lib、/usr/lib64、/lib、/lib64）；
  • 将库路径增加到环境变量中：在终端输入export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:xxx，其中 xxx 为动态链接库文件的绝对存储路径（此方式仅在当前终端有效，关闭终端后无效）；或修改~/.bashrc 或~/.bash_profile 文件，即在文件最后一行添加export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:xxx（xxx 为动态库文件的绝对存储路径）。保存之后，执行source .bashrc指令（此方式仅对当前登陆用户有效）。
  • 配置 /etc/ld.so.conf 文件，在该文件中增加动态链接库的绝对路径，然后执行命令 ldconfig -v，更新配置。
