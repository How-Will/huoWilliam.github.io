fastDFS简介

FastDFS 是一款开源的、分布式文件系统（Distributed File System），通过纯 C 实现。系统架构如图所示：

文件上传的过程如下

  1. 首先客户端 client 发起对 FastDFS 的文件传输动作，是通过连接到某一台 Tracker 的指定端口来实现的；
  2. 然后 Tracker 根据目前掌握的信息，来决定选择哪一台 storage，然后将该 storage 的地址等信息返回给 client。
  3. 最后 client 通过这些信息连接到该 storage，进行 TCP 文件传输。

FastDFS服务端有两个重要角色：tracker 和 storage：

  • tracker：主要做调度工作，在访问上起到负载均衡的作用
  • storage：存储文件，完成文件管理的所有功能

tracker 和 storage 之间不直接通信，其基本的信息由配置文件在系统启动加载时获知。

storage 可以分成多个 Group，每个 Group 里面的 storage 是相互备份的关系【Group 的存储能力由该 Group 内存储能力最小的 storage 决定，纵向扩展】。所以要提高文件系统的总体存储容量，需要添加新的 Group【横向扩展】。

tracker 可以有多台，每台 tracker 之间是平等的，形成分布式，防止单点故障问题【高可靠性】。
